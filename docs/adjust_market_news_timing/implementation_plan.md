# マーケットニュース表示タイミング変更計画（改定版）

ユーザーの要望により、1ターン目のマーケットニュース表示タイミングを「ROLLボタン押下後」から「ROLLボタン押下前（ターン開始時）」に変更します。これにより、ゲーム開始直後にニュースが表示されている状態になります。

## 意図の確認

現在の実装（`rollDice` 関数内でのニュース呼び出し）は、プレイヤーのアクション（ROLL）に対してニュース演出を割り込ませる意図で作られていると考えられます。しかし、これを「ターン開始時に既にニュースが出ている」状態に変更することで、プレイヤーがニュースを確認してからダイスを振れるようにします。

## 変更内容

### `src/store/gameStore.ts`

- **[MODIFY]** `rollDice` アクション内のニューストリガーロジックを削除します。
  - `if (turnCount === 1 && activePlayerIndex === 0 && !get().hasRolled && get().currentNews === null)` のブロックを削除。

### `src/components/Board.tsx`

- **[MODIFY]** `useEffect` を追加し、コンポーネントのマウント時（またはターン開始時）に以下の条件でニュースをトリガーします。
  - `turnCount === 1`
  - `activePlayerIndex === 0`
  - `!currentNews`
  - `!hasRolled`

この変更により、画面が表示された時点で（またはターンが回ってきた時点で）自動的にニュースが表示されるようになります。

## 検証方法

1. アプリケーションをリロードし、新しいゲームを開始する。
2. 画面が表示された直後（または少し遅れて）にマーケットニュースが表示されることを確認する。
3. ニュースを閉じた後、通常通り「ROLL」ボタンが押せることを確認する。
4. 2ターン目以降の挙動に影響がないか確認する（一周した後に再度ニュースが出るか確認）。
